Project Blueprint: AI Auto-Blogging Engine
Subtitle: RSS + News API → SEO-Optimized Blog Publisher
1. Project Objective
Build a scalable, SaaS-ready web application that automatically generates and publishes SEO-optimized blog posts for users running websites on WordPress, Webflow, or custom stacks (PHP, Next.js, Node.js). The system fetches content from RSS feeds/News APIs, intelligently rewrites and structures it using AI to avoid plagiarism, generates/attaches images, and auto-publishes at user-defined scheduled intervals.
2. Technology Stack
Backend: Python (FastAPI) - For high performance, async API endpoints.
Frontend: Next.js (React, TypeScript, Tailwind CSS) - Dashboard UI.
Database: PostgreSQL (using SQLAlchemy or SQLModel ORM).
Task Queue & Scheduling: Celery with Redis broker - Crucial for handling long-running AI tasks and cron-like publishing schedules.
AI Integration: Modular Provider System (OpenAI API, Google Gemini API).
Deployment: Docker & Docker Compose (containerized microservices).
3. Core Architecture & Workflow
3.1. User Setup Phase
Auth & Keys: User registers, inputs API keys for AI (OpenAI/Gemini) and CMS (WordPress App Passwords, Webflow API tokens).
Project Creation: User creates a "Project" defining their niche (e.g., "Crypto News").
Source Configuration: User adds RSS feed URLs or selects News APIs (e.g., NewsAPI.org).
Keyword Strategy: User defines primary keywords, secondary keywords, and target tone.
Scheduling: User configures posting frequency (e.g., "Publish 2 posts every day at 09:00 and 15:00 UTC").
3.2. Execution Engine (Automated Loop)
Ingestion (Cron/Celery Task): Fetch latest articles from configured RSS/News APIs.
Filtering & Deduplication: Check DB to ensure the article hasn't been processed. Filter out non-relevant articles based on keywords.
AI Transformation Pipeline:
Step 1 (Extraction): Extract core facts and narrative from the source text.
Step 2 (Drafting): AI writes a completely original, 800–1500 word article based only on the facts (ensuring 0% plagiarism).
Step 3 (SEO Formatting): Inject H1/H2/H3 tags, keywords, and natural density.
Step 4 (Metadata): Generate SEO Title, Meta Description, and clean URL slug.
Media Processing: Attempt to extract the primary image from the source URL. If unavailable or invalid, trigger AI Image Generation (DALL-E 3 / Imagen) based on the article's context.
CTA Injection: Append user-defined Call-To-Action blocks dynamically.
Publishing (Delivery): Push the final payload via REST API (WordPress), CMS API (Webflow), or standard Webhook (Custom Next.js/Node backends).
4. Detailed Feature Specifications
Feature 1: Source Integration Engine
RSS Parser: Use standard Python libraries (e.g., feedparser) to parse XML/Atom feeds.
News API Adapters: Built-in connectors for standard news APIs.
Hash Deduplication: Create an MD5 hash of the source article URL and title to prevent duplicate processing.
Feature 2 & 9: AI Content & Quality Engine (CRITICAL)
Strict Anti-Plagiarism Protocol: The prompt chain MUST forbid direct copying. The system should summarize the source into bullet points, then instruct the AI to write a brand new article from those bullet points in a specific tone.
Readability & Flow: Prompt engineering must enforce short paragraphs, active voice, and transition words to maintain a human-like, engaging tone.
Feature 3: SEO Formatting Engine
Output Schema: The AI must return structured JSON containing:
{
  "seo_title": "String (max 60 chars)",
  "meta_description": "String (max 160 chars)",
  "slug": "kebab-case-string",
  "content_html": "Full HTML string with <h2>, <h3>, <p>, <ul>",
  "suggested_tags": ["tag1", "tag2"]
}

Keyword Enforcement: Post-generation script validates that primary keywords appear in the H1, at least one H2, and the meta description.
Feature 4: Image Management System
Extraction: Use libraries like newspaper3k or BeautifulSoup to scrape og:image.
Generation: Fallback to OpenAI Image generation API.
Optimization: Ensure alt-text is generated by the AI based on the image description.
Feature 5: Scheduling & Task Queue (Celery)
Queue Structure:
fetch_queue: Polls RSS feeds every X hours.
ai_generation_queue: Handles the heavy API calls to OpenAI/Gemini.
publish_queue: Executes the final HTTP requests to the user's CMS.
Retry Logic: Exponential backoff (e.g., retry publishing after 5 mins, then 15 mins) for CMS timeouts.
Feature 6: CMS Integrations
WordPress: Use the WP REST API (/wp-json/wp/v2/posts). Handle basic auth (Application Passwords). Map generated HTML to the content field, Title to title, and categories/tags.
Webflow: Use Webflow CMS API (POST /collections/{collection_id}/items). Map JSON fields to user-defined Webflow CMS schema.
Custom (Webhook): Send a POST request with a predefined JSON payload to a user-provided endpoint URL.
Feature 7: CTA System
Template Builder: UI allows users to create standard HTML/Markdown snippets (e.g., "Ready to invest? Click here").
Injection Logic: Automatically append this snippet to the end of the content_html before publishing.
5. Conceptual Database Schema (PostgreSQL)
Users: id, email, hashed_password, created_at
Projects: id, user_id, name, cms_type (wp/webflow/custom), cms_credentials (encrypted JSON), cta_template
Sources: id, project_id, type (rss/news_api), url, is_active
Generation_Settings: id, project_id, primary_keywords, niche, tone, ai_provider, encrypted_ai_api_key
Schedules: id, project_id, frequency_hours, next_run_time
Posts (Audit Log): id, project_id, source_url, generated_title, status (pending/generated/published/failed), published_url, error_log
6. Implementation Phases for AI/Cursor
Phase 1: Foundation (Backend & DB)
Set up FastAPI, SQLAlchemy, and PostgreSQL Docker containers.
Define database models and CRUD endpoints.
Phase 2: Ingestion & AI Pipeline
Implement RSS fetching (feedparser).
Build the AI integration layer using LangChain or direct provider SDKs (OpenAI/Gemini).
Create the "Anti-Plagiarism" extraction & rewriting prompt chain.
Phase 3: Queues & Workers
Integrate Redis and Celery.
Create async tasks for fetching, generating, and publishing.
Phase 4: CMS Integrations
Build the WordPress REST API publisher.
Build the Webflow CMS API publisher.
Build the generic Webhook publisher.
Phase 5: Frontend Dashboard
Initialize Next.js project.
Build User Dashboard (Connect APIs, Add RSS feeds, View generated logs).
Implement scheduling UI.
7. Crucial Security & Privacy Rules
API Key Encryption: All user-provided API keys (OpenAI, WP Passwords) MUST be encrypted in the database using a robust symmetric encryption library (e.g., cryptography.fernet).
No Data Lock-in: Ensure users can easily delete their project data and API keys.
Rate Limiting: Implement strict rate limits on the FastAPI routes to prevent abuse.
